## 1. Component Overview

### 1.1 Purpose

The operator rebuild authorization boundary enforces that **only designated operator users** can execute destructive graph rebuild operations. This is critical because rebuild operations are:

- **Destructive**: They replace the entire graph state
- **State-mutating**: They modify persisted truth
- **Operationally sensitive**: Failed rebuilds can impact production
- **Persistence-affecting**: They write to durable storage

### 1.2 Architecture

The authorization boundary is implemented as a FastAPI dependency injection:

```python
# api/auth.py
def get_current_rebuild_operator_user(
    current_user: Annotated[User, Depends(get_current_active_user)],
    settings: Annotated[Settings, Depends(get_settings)],
) -> User:
    """
    Enforce operator authorization for destructive graph rebuild actions.

    Allows only the configured admin username from settings; denies other active
    users with a bounded forbidden response.
    """
```

### 1.3 Protected Endpoints

The following endpoints are protected by operator authorization:

1. **`POST /api/graph/rebuild`** — Trigger graph rebuild and persistence
2. **`GET /api/graph/rebuild/jobs/{job_id}`** — Read rebuild job details
3. **`GET /api/graph/rebuild/jobs`** — List rebuild jobs
4. **`POST /api/graph/rebuild/jobs/{job_id}/cancel`** — Request cancellation of a pending or running rebuild job

All four endpoints require `Depends(get_current_rebuild_operator_user)`.

## 2. Implementation Analysis

### 2.1 Core Authorization Logic

**Location**: `api/auth.py:428-451`

**Implementation**:

```python
def get_current_rebuild_operator_user(
    current_user: Annotated[User, Depends(get_current_active_user)],
    settings: Annotated[Settings, Depends(get_settings)],
) -> User:
    configured_admin = (settings.admin_username or "").strip()

    if not configured_admin:
        raise HTTPException(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            detail=REBUILD_OPERATOR_NOT_CONFIGURED_DETAIL,
        )

    if current_user.username.strip() != configured_admin:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail=REBUILD_OPERATOR_FORBIDDEN_DETAIL,
        )
    return current_user
```

**Strengths**:

- ✅ Bounded error messages (no secret leakage)
- ✅ Defensive null/whitespace handling
- ✅ Centralized settings dependency
- ✅ Appropriate HTTP status codes (503 for misconfiguration, 403 for unauthorized)
- ✅ Fails closed when configuration is missing

**Assessment**: **EXCELLENT** — Production-quality implementation.

### 6.1 Current Architecture

The current implementation uses **username-based operator check**:

```python
if current_user.username.strip() != configured_admin:
    raise HTTPException(status_code=status.HTTP_403_FORBIDDEN, ...)
```

### 6.2 RBAC Migration Path

**Future State** (conceptual):

```python
def get_current_rebuild_operator_user(
    current_user: Annotated[User, Depends(get_current_active_user)],
    rbac_service: Annotated[RBACService, Depends(get_rbac_service)],
) -> User:
    if not rbac_service.has_permission(current_user, "graph:rebuild"):
        raise HTTPException(status_code=status.HTTP_403_FORBIDDEN, ...)
    return current_user
```

**Migration Strategy**:

1. Introduce RBAC service
2. Map `ADMIN_USERNAME` to "operator" role in RBAC
3. Update `get_current_rebuild_operator_user` to check RBAC permissions
4. Maintain backward compatibility during transition
5. Deprecate `ADMIN_USERNAME`-based check after RBAC proven

**Assessment**: ✅ Current implementation is **abstracted behind dependency injection**, making RBAC migration straightforward.
