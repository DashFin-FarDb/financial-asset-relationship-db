### Bootstrap local companion

`npx -y snipara-companion@latest init --client codex --project context-free`

Re-run local companion setup if this workstation needs to refresh browser login or project selection.

### Run doctor for this project

`create-snipara doctor`

Inspect local wiring, hosted reachability, and generated assets.

### Repair local setup

`create-snipara repair`

Rebuild config files, hooks, and the companion pack.

### Run a focused hosted query

`npx -y snipara-companion@latest query --query "how does auth middleware work?" --max-tokens 4000`

Pull ranked source context before editing, reviewing, or drafting.

### Load linked standards only when needed

`npx -y snipara-companion@latest shared-context --categories MANDATORY BEST_PRACTICES`

Fetch project-linked team or workspace guidance without treating it as memory.

### Generate a hosted plan

`npx -y snipara-companion@latest plan --query "Implement webhook retry hardening" --max-tokens 3000`

Produce a structured execution plan against indexed project knowledge.

### Start a managed workflow from the visible plan

`npx -y snipara-companion@latest workflow start --goal "Implement webhook retry hardening" --plan-file ./plan.json`

Record the JSON machine plan in .snipara/workflow/current.json so stable phase ids survive compaction.

### Start the current workflow phase

`npx -y snipara-companion@latest workflow phase-start context`

Mark a phase active and print the required Snipara context gate for the agent.

### Analyze code impact before a risky change

`npx -y snipara-companion@latest code impact --changed-files src/api/webhooks.ts tests/webhooks.test.ts --diff-summary "webhook retry hardening"`

Use paid Context code impact analysis to surface likely tests, docs, routes, config, and risks.

## Design Rule

- Context first: use hosted queries, shared-context, and document loads for source truth.
- Memory second: use session bootstrap and task commit for durable decisions, learnings, and preferences.
- Managed workflow when a visible plan exists: keep the machine plan in JSON, then use `workflow start`, `phase-start`, `phase-commit`, `workflow resume`, and `final-commit` to keep phase state durable across compaction. For runtime-bound phases, use `workflow runtime-checkpoint` so resume can print a reattach or rehydrate path.
- Durable memory, policy, and orchestration stay hosted in Snipara.
- The local companion is optional tooling for capture, bootstrap, upload, exploration, doctor, repair, and explicit task summaries.
