# Elastic Copilot Memory Files

This directory contains AI memory files generated by the Elastic Copilot tool. These files help maintain context and project knowledge across AI-assisted development sessions.

## Files

### systemManifest.md

The system manifest provides a current snapshot of the project structure, dependencies, and key components.

**Important**: This file should contain only **one occurrence** of each section heading. Duplicate sections are automatically detected and prevented through CI validation.

#### Standard Sections

The manifest includes the following level 2 (##) sections:

1. **Project Overview** - Basic project information (name, description, created date)
2. **Current Status** - Current development phase and last update timestamp
3. **Project Structure** - File counts by type (py, js, ts, tsx files)
4. **Dependencies** - Overview of dependency management
5. **Project Directory Structure** - Tree view of the project file structure
6. **PY Dependencies** - Python dependencies by file
7. **JS Dependencies** - JavaScript dependencies by file
8. **TS Dependencies** - TypeScript dependencies by file
9. **TSX Dependencies** - TypeScript React dependencies by file
10. **Key Components** - Important architectural components
11. **Integration Points** - System integration information
12. **Technical Considerations** - Technical notes and constraints
13. **Implementation Notes** - Implementation-specific details

Each section should appear exactly **once** in the manifest.

## Maintenance Scripts

### Deduplication Script

If duplicate sections accumulate in the manifest (typically from repeated AI tool runs), use the deduplication script:

```bash
python scripts/deduplicate_manifest.py
```

This script:
- Parses the manifest into sections
- Identifies duplicate section headings
- Keeps only the **last occurrence** of each section (most recent)
- Creates a backup at `.elastic-copilot/memory/systemManifest.md.backup`
- Writes the deduplicated version back to the original file

**When to use**: Run this script if you notice:
- The manifest file growing unexpectedly large (>1000 lines)
- Multiple occurrences of the same section heading
- CI validation failures due to duplicate headings

### Validation Script

To check for duplicate sections without making changes:

```bash
python scripts/validate_manifest.py
```

This script:
- Scans for duplicate level 2 (##) headings
- Reports the number of duplicates and their line numbers
- Exits with code 0 if clean, 1 if duplicates found
- Used by CI to prevent duplicate sections from being merged

**CI Integration**: This validation runs automatically on:
- Pull requests that modify `systemManifest.md`
- Pushes to the main branch affecting the manifest

## Section Contract

### Section Heading Format

Level 2 headings must follow this format:
```markdown
## Section Name
```

**Rules**:
- Must start with exactly `## ` (two hashes and a space)
- Must not be a level 3 heading (`###` or more)
- Each heading should be unique within the file
- Section content continues until the next level 2 heading

### Content Guidelines

- **Keep sections current**: Update existing sections rather than appending new ones
- **Single source of truth**: Each section should reflect the current state of the project
- **Structured format**: Follow established patterns for each section type
- **Regular maintenance**: Run deduplication after significant AI tool usage

## CI Validation (MD024)

The repository enforces the **MD024 markdownlint rule** (no-duplicate-heading) specifically for `systemManifest.md`.

Configuration is in `.markdownlint.json`:
```json
{
  "MD024": {
    "siblings_only": false
  }
}
```

This ensures:
- No duplicate section headings at any level
- All duplicates are caught before merge
- Manifest remains clean and usable

## Troubleshooting

### "Duplicate heading found" error in CI

**Cause**: Multiple occurrences of the same section heading exist in the manifest.

**Solution**:
```bash
# 1. Run deduplication script
python scripts/deduplicate_manifest.py

# 2. Verify the fix
python scripts/validate_manifest.py

# 3. Commit the cleaned manifest
git add .elastic-copilot/memory/systemManifest.md
git commit -m "chore: deduplicate systemManifest.md sections"
```

### Manifest file is too large (>5000 lines)

**Cause**: Sections have been appended repeatedly instead of replaced.

**Solution**: Same as above - run the deduplication script.

### Need to restore from backup

If deduplication causes issues, restore from backup:
```bash
cp .elastic-copilot/memory/systemManifest.md.backup .elastic-copilot/memory/systemManifest.md
```

## Best Practices

1. **Check before committing**: Run validation before committing manifest changes
2. **Deduplicate regularly**: Run the deduplication script if the file exceeds 500 lines
3. **Review changes**: Always review manifest changes in pull requests
4. **Keep content current**: Update sections in place rather than appending
5. **Use CI feedback**: Address validation errors before requesting review
